╚══════════════════════════════════════════════════════════════════════════════╝

┌──────────────────────────────────────────────────────────────────────────────┐
│  Problème: "Cannot connect to localhost:8084"                               │
└──────────────────────────────────────────────────────────────────────────────┘

Cause: Le service n'est pas démarré

Solution:
1. Vérifier que le service tourne dans IntelliJ
2. Regarder les logs (erreurs en rouge)
3. Redémarrer le service

┌──────────────────────────────────────────────────────────────────────────────┐
│  Problème: "Access denied for user 'root'@'localhost'"                      │
└──────────────────────────────────────────────────────────────────────────────┘

Cause: Mauvais mot de passe MySQL

Solution:
1. Ouvrir: src/main/resources/application.properties
2. Modifier la ligne:
   spring.datasource.password=votre_mot_de_passe
3. Redémarrer le service

┌──────────────────────────────────────────────────────────────────────────────┐
│  Problème: "Port 8084 already in use"                                       │
└──────────────────────────────────────────────────────────────────────────────┘

Cause: Un autre processus utilise le port 8084

Solution 1: Tuer le processus
   netstat -ano | findstr :8084
   taskkill /PID <PID> /F

Solution 2: Changer le port
   Dans application.properties:
   server.port=8085

┌──────────────────────────────────────────────────────────────────────────────┐
│  Problème: MySQL ne démarre pas                                             │
└──────────────────────────────────────────────────────────────────────────────┘

Solution:
1. Si vous avez XAMPP: Ouvrir XAMPP Control Panel → Start MySQL
2. Sinon: net start MySQL80
3. Vérifier que MySQL tourne sur le port 3306
